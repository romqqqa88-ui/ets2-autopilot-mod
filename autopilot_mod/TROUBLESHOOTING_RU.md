# Устранение неполадок мода автопилота для ETS 2

Если мод не работает, следуйте этим инструкциям для диагностики и решения проблем.

## 🔍 Быстрая диагностика

### 1. Проверка установки
- [ ] Мод скопирован в папку `Documents/Euro Truck Simulator 2/mods/`
- [ ] Файл имеет расширение `.scs` или находится в папке
- [ ] Мод активирован в меню "Моды" игры

### 2. Проверка версии игры
- Откройте консоль игры (клавиша `~`)
- Введите `g_version`
- Убедитесь, что версия 1.58 (мод поддерживает 1.52-1.58)

### 3. Проверка консоли на ошибки
- Откройте консоль игры (`~`)
- Ищите сообщения с `[Autopilot]` или `[Simple Autopilot]`
- Ищите ошибки (красный текст)

## 🐛 Распространенные проблемы и решения

### Проблема 1: Мод не появляется в меню модов
**Причины:**
- Неправильное расположение файлов
- Поврежденный архив мода
- Конфликт с другими модами

**Решение:**
1. Убедитесь, что структура папки `autopilot_mod/` содержит:
   - `manifest.sii`
   - `init.lua` 
   - `def/`, `lua/`, `material/` подпапки
2. Попробуйте создать архив `.scs`:
   - Запустите `build_mod.bat`
   - Используйте полученный `autopilot_mod.scs`

### Проблема 2: Мод активирован, но не работает
**Причины:**
- Отсутствие сообщений в консоли
- Конфликт Lua окружения
- Ошибки в скриптах

**Решение:**
1. Проверьте консоль на наличие сообщений инициализации
2. Запустите диагностический скрипт:
   ```bash
   lua diagnose_mod.lua
   ```
3. Попробуйте упрощенную версию:
   - Временно замените `init.lua` на `simple_autopilot.lua`
   - Переименуйте `simple_autopilot.lua` в `init.lua`

### Проблема 3: Горячие клавиши не работают
**Причины:**
- Конфликт клавиш с другими модами
- Ошибка регистрации обработчиков
- Неправильные коды клавиш

**Решение:**
1. Проверьте консоль на сообщения о регистрации горячих клавиш
2. Попробуйте альтернативные клавиши:
   - В консоли введите `autopilot` для включения/выключения
   - Используйте `autopilot_status` для проверки состояния
3. Измените клавиши в коде (в `init.lua` или `simple_autopilot.lua`)

### Проблема 4: Игра вылетает при загрузке
**Причины:**
- Критические ошибки в Lua коде
- Конфликт памяти
- Несовместимость с другими модами

**Решение:**
1. Отключите все другие моды, оставьте только автопилот
2. Проверьте файл `game.log.txt` в папке игры
3. Упростите мод:
   - Используйте только `simple_autopilot.lua`
   - Удалите сложные модули

## 🧪 Пошаговая проверка работы

### Шаг 1: Базовая проверка
1. Запустите игру с активированным модом
2. Откройте консоль (`~`)
3. Проверьте наличие сообщений:
   ```
   [Autopilot Mod] Инициализация мода автопилота для ETS 2
   [Autopilot Mod] Мод загружен, начинаю инициализацию...
   ```

### Шаг 2: Проверка инициализации
1. Дождитесь полной загрузки игры
2. Проверьте консоль на сообщения:
   ```
   [Autopilot Mod] Мод успешно инициализирован и готов к работе
   [Autopilot Mod] Используйте NumPad 0 для включения/выключения автопилота
   ```

### Шаг 3: Проверка функциональности
1. Нажмите `NumPad 0` - должно появиться сообщение "Автопилот ВКЛЮЧЕН"
2. Нажмите `NumPad 1` - должно появиться сообщение о статусе
3. В консоли введите `autopilot_status`

## 🔧 Расширенная диагностика

### 1. Проверка файла game.log
Файл `game.log.txt` находится в:
- Windows: `Documents/Euro Truck Simulator 2/game.log.txt`

Ищите в нем:
- `Loading mod` - загрузка мода
- `Error` или `Failed` - ошибки
- `autopilot` - сообщения мода

### 2. Тестирование вне игры
Запустите тестовые скрипты:
```bash
# Проверка структуры
lua diagnose_mod.lua

# Запуск тестов совместимости  
lua test_runner.lua

# Проверка упрощенной версии
lua -e "dofile('simple_autopilot.lua')"
```

### 3. Создание логов мода
Мод создает логи в `logs/autopilot.log` (если включено логгирование).
Проверьте этот файл после запуска игры.

## 🛠️ Альтернативные варианты

### Вариант A: Упрощенный мод
Если полная версия не работает:
1. Удалите все файлы кроме:
   - `manifest.sii`
   - `simple_autopilot.lua` (переименуйте в `init.lua`)
   - `def/autopilot/config.sii`
2. Создайте новый архив `.scs`

### Вариант B: Минимальный тестовый мод
Создайте минимальный рабочий мод для проверки:
1. Создайте папку `test_autopilot/`
2. Добавьте только `manifest.sii` и `init.lua` с простым кодом
3. Протестируйте его отдельно

## 📞 Получение помощи

Если проблемы остаются:

1. **Предоставьте информацию:**
   - Версия игры (из консоли)
   - Содержимое `game.log.txt` (первые 50 строк после ошибки)
   - Сообщения из консоли игры
   - Список активированных модов

2. **Проверенные решения:**
   - Обновите игру до последней версии
   - Проверьте целостность файлов игры через Steam
   - Отключите все другие моды
   - Переустановите мод

3. **Дополнительные ресурсы:**
   - Файл `README.md` - общее описание
   - Файл `INSTALL_RU.md` - инструкция по установке
   - Папка `logs/` - логи работы мода

## ✅ Признаки успешной работы

Мод работает правильно, если:
- [ ] В консоли появляются сообщения инициализации
- [ ] Нажатие NumPad 0 меняет состояние автопилота
- [ ] В консоли отображается статус по команде `autopilot_status`
- [ ] Создается файл `logs/autopilot.log` (если включено логгирование)
- [ ] Игра не вылетает и работает стабильно
